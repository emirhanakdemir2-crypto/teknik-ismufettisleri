# Resmî Teknik Kaynaklar

Agent, sürüme bağlı veya güvenlik açısından önemli bir teknik konuda tahmin yürütmeden önce ilgili resmî kaynağı kontrol etmelidir.

## Next.js

- Dokümantasyon: https://nextjs.org/docs
- Kullanım: App Router, Server Components, Server Actions, Route Handlers ve `proxy.ts`

## Supabase

- Dokümantasyon: https://supabase.com/docs
- Kullanım: Auth, SSR, PostgreSQL, RLS, Storage ve Realtime

## Vercel

- Dokümantasyon: https://vercel.com/docs
- Kullanım: Deployment, environment variables, functions ve cron

## Upstash

- Dokümantasyon: https://upstash.com/docs
- Kullanım: Redis ve server-side rate limiting

## Resend

- Dokümantasyon: https://resend.com/docs
- Kullanım: Transactional email

## Cursor

- Dokümantasyon: https://cursor.com/docs
- Kullanım: Rules, Agent, Skills ve MCP

## Kaynak Politikası

- Öncelikle resmî dokümantasyon kullanılmalıdır.
- Eski blog yazıları source of truth kabul edilmemelidir.
- Versiyona bağlı konularda mevcut `package.json` kontrol edilmelidir.
- Dokümantasyonla mevcut proje sürümü çelişirse işlem durdurulup raporlanmalıdır.
- Yeni bağımlılık eklenmeden önce mevcut stack ile çözüm değerlendirilmelidir.
